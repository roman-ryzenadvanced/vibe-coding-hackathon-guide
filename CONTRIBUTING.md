# Contributing to Vibe Coding Builders Hackathon Guide

Thank you for your interest in contributing! This guide welcomes additions from the community.

## How to Contribute

### Add a New Hackathon Idea

1. Fork this repository
2. Create a new feature doc in `features/idea-NN-name.md` following the template below
3. Update `README.md` to include the new idea in the table
4. Update `FEATURES.md` with the skill module mapping
5. Update `CHANGELOG.md` with your addition
6. Submit a pull request

### Idea Template

```markdown
# Idea NN: [Name]

## Quick Stats
- **Difficulty**: Beginner / Intermediate / Advanced
- **Time to Build**: X days
- **Core Skill Feature**: [which last30days feature]
- **API Keys Required**: None / List required keys

## The Problem
[What real-world problem does this solve? Reference signal data if possible.]

## The Solution
[How does this project address the problem? Be specific about the approach.]

## Architecture
[Technical architecture diagram or description]

## Data Flow
1. [Step 1]
2. [Step 2]
3. ...

## Skill API Mapping
| Skill Feature | How It's Used |
|---------------|--------------|
| [feature] | [usage] |

## Tech Stack
- Frontend: [framework]
- Backend: [framework]
- Data: [source]

## Implementation Steps
1. [Step 1 with time estimate]
2. [Step 2 with time estimate]

## Demo Script
[What to show in a 3-minute demo]
```

### Improve Documentation

- Fix typos or unclear instructions
- Add screenshots or diagrams
- Improve the build process
- Add new prompt examples

### Report Issues

- Open a GitHub issue with:
  - Description of the issue
  - Steps to reproduce (if applicable)
  - Expected vs. actual behavior
  - Screenshots (if visual)

## Style Guide

- Use dark mode aesthetic in all visual assets
- Follow the Z.ai + rommark.dev branding colors
- Keep descriptions concise but complete
- Include difficulty ratings for all ideas
- Reference specific skill commands and features

## Code of Conduct

- Be respectful and constructive
- Focus on building, not tearing down
- Share knowledge openly
- Credit original authors and sources
