## The Problem

In today's hyper-competitive SaaS landscape, staying informed about what your competitors are doing is not optional — it's existential. Yet most founders and product teams rely on manual, ad-hoc competitive intelligence: someone occasionally checks a competitor's blog, a sales rep forwards a feature comparison email, or a founder stumbles upon a competitor's ProductHunt launch after it's already gone viral. This reactive approach means companies are always a step behind. Competitors ship features you didn't see coming, pivot into your niche without warning, and capture market share while you're still reading last month's changelog. The problem is compounded for indie hackers and small teams who don't have dedicated competitive intelligence analysts or enterprise-grade tools like Crayon or Klue that cost thousands per month. Even when competitive data is available, it's scattered across dozens of sources — GitHub repos, pricing pages, job postings, social media announcements, review sites — making it nearly impossible to synthesize into actionable insights. The lack of real-time, automated competitive monitoring means decisions are made on stale information, opportunities are missed, and threats are recognized too late. For hackathon builders, this problem is even more acute: you need to understand the competitive landscape of your idea within hours, not weeks, to avoid building something that already exists or entering a space with insurmountable incumbents.

---

## The Solution

The Competitor Intelligence Dashboard provides real-time, automated A/B/C comparison of any set of competitors using the last30days skill's Comparison Mode and Watchlist features. Instead of manually checking each competitor's website, social media, and changelog, you define your competitive set once, and the system continuously monitors all relevant signals — product updates, pricing changes, hiring patterns, social sentiment, feature launches, and customer complaints. The Comparison Mode engine generates structured side-by-side comparisons with visual diff highlights, showing exactly what changed and when. The Watchlist feature enables Slack digest delivery, so your team receives a daily or weekly summary of competitive movements without ever leaving their workflow. The dashboard presents a unified view with three core sections: the Competitive Radar (a visual map of competitive positioning that updates in real-time), the Change Feed (a chronological log of detected competitor movements with source links), and the Comparison Matrix (a feature-by-feature comparison table with automated gap analysis). The system uses entity resolution to track competitors across naming variations and the cluster merge algorithm to group related signals into coherent narratives. For hackathon teams, this means you can validate your competitive advantage in minutes and continuously monitor for threats throughout the build weekend.

## Architecture

The Competitor Intelligence Dashboard is built as a three-tier system: Data Collection, Intelligence Processing, and Presentation. The Data Collection tier uses the last30days skill's signal scanner to continuously poll competitor-related sources including product pages, GitHub repositories, social media accounts, job boards, and review sites. Each competitor is defined as a "watched entity" with associated source URLs and keywords. The Intelligence Processing tier applies entity resolution to unify mentions across sources, cluster merge to group related signals, and a comparison engine that generates structured A/B/C comparisons with diff detection. The comparison engine tracks features, pricing tiers, positioning language, and customer sentiment over time, flagging any delta as a "competitive movement." The Presentation tier delivers insights through three channels: a web dashboard (Next.js with real-time updates via Server-Sent Events), Slack digests (via Watchlist webhook integration), and HTML briefs (generated by the briefing engine for email or print). A cron-based scheduler triggers scans at configurable intervals (hourly, daily, weekly) and pushes updates to the Watchlist subscribers.

```
[Competitor Sources] → Signal Scanner → Entity Resolution → Cluster Merge
                                                              ↓
[Slack Digest] ← Watchlist Dispatcher ← Comparison Engine ← Change Detector
                                                              ↓
[Web Dashboard] ← SSE Updates ← Intelligence Store ← Movement Logger
```

---

## Data Flow

1. User defines competitor set via dashboard or `/watch add CompetitorA CompetitorB CompetitorC` command.
2. Signal Scanner polls each competitor's known sources (website, GitHub, social, reviews) on schedule.
3. Raw signals are normalized into unified schema: `{ entity, source_type, content, timestamp, url, sentiment }`.
4. Entity Resolution maps signals to the correct competitor regardless of naming variations or abbreviations.
5. Cluster Merge groups related signals (e.g., a GitHub PR, a blog post, and a tweet about the same feature launch).
6. Comparison Engine runs structured comparison across the competitor set on dimensions: features, pricing, positioning, sentiment, velocity.
7. Change Detector identifies deltas from the previous scan and flags them as competitive movements.
8. Movement Logger persists all changes with timestamps for historical trend analysis.
9. Watchlist Dispatcher pushes formatted digest to Slack channel via webhook.
10. Dashboard receives SSE updates and renders real-time competitive radar, change feed, and comparison matrix.

## Tech Stack

- **Frontend**: Next.js 16, TypeScript, Tailwind CSS 4, shadcn/ui, Recharts for radar visualizations
- **AI/Analysis**: z.ai GLM API for sentiment analysis and narrative generation
- **Signal Processing**: last30days skill (signal scanner, comparison mode, watchlist, entity resolution)
- **Real-time Updates**: Server-Sent Events (SSE) for live dashboard updates
- **Notifications**: Slack Webhook API for digest delivery
- **Database**: Prisma ORM with PostgreSQL for persistent competitive data and movement history
- **Scheduling**: node-cron for configurable scan intervals
- **Deployment**: Vercel with cron jobs support
