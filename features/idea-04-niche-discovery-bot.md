## The Problem

The hardest part of building a successful indie product isn't coding — it's finding the right niche. Most indie hackers spend weeks or months in "idea limbo," bouncing between concepts that feel promising but lack validation. They browse Indie Hackers, scroll ProductHunt, and ask friends for feedback, but this approach is fundamentally flawed: it's slow, biased, and lacks the data-driven rigor needed to separate genuine opportunities from mirages. The statistics are sobering — the vast majority of indie projects fail not because of poor execution, but because they solve problems nobody has or enter markets that are already saturated. Meanwhile, the signals that indicate a profitable niche — repeated complaints with high engagement, growing search volume, low competitive density, and willingness to pay — are scattered across dozens of platforms and require systematic analysis to detect. Discord communities where indie hackers gather are full of people asking "What should I build?" but providing anecdotal, unvalidated answers. There's no structured, interactive process that guides someone from "I have general interests" to "Here's a validated niche with quantified opportunity." The gap between curiosity and conviction is where most aspiring founders give up. What's needed is not another list of "101 startup ideas" but a guided, data-driven discovery process that adapts to each person's skills, interests, and constraints — like having a YC partner in your pocket who actually knows the market data.

---

## The Solution

The Niche Discovery Bot is a Discord bot that uses the last30days skill's Interactive Gate pattern and Signal Scanner to guide indie hackers through a structured, data-driven niche discovery workflow. Described as "YC in a Discord bot," it replaces the aimless brainstorming process with a guided journey that starts broad and narrows progressively based on both user input and real-time market signals. The bot uses the Interactive Gate pattern — a conversational decision tree where each gate asks a targeted question and uses the answer to refine the search space. Gate 1: "What verticals interest you?" Gate 2: "What's your technical comfort zone?" Gate 3: "B2B or B2C?" Gate 4: "What's your timeline?" After each gate, the bot runs a signal scan to validate or invalidate the narrowing direction, presenting real data (engagement scores, competitive density, monetization signals) rather than opinions. Once all gates are passed, the bot delivers a Niche Report: a ranked list of validated opportunities with Signal Scores, user quotes, competitive landscape, and a 30-day launch plan. The bot operates entirely within Discord, making it accessible to the communities where indie hackers already spend their time. It can also be invoked in public channels, creating a shared discovery experience where community members can collaborate on validation. For hackathon teams, the bot can compress the entire discovery process from weeks to minutes, letting you start building with confidence by Friday night.

## Architecture

The Niche Discovery Bot is built as a Discord bot with a stateful conversation engine, four Interactive Gate stages, and a signal processing backend. The bot runs on a Node.js process using discord.js for Discord API integration. The conversation engine maintains per-user state (current gate, answers, constraints) in a SQLite database. Each Interactive Gate is implemented as a structured prompt-response pair: the bot sends a question with interactive buttons or emoji reactions, the user responds, and the engine updates the user's profile and triggers a signal scan with the refined parameters. The signal processing backend uses the last30days skill's signal scanner, entity resolution, and cluster merge to validate each narrowing step. If the signal scan returns low-opportunity results for the user's constraints, the bot proactively suggests pivots with supporting data. The final Niche Report is generated by the GLM API, which synthesizes all gate answers and signal data into a structured opportunity assessment with actionable recommendations. The bot supports both DM (private discovery) and channel (public discovery) modes.

```
[Discord User Input] → Interactive Gate Engine → User Profile Store
                              ↓                        ↓
[Signal Scanner] ← Gate Parameter Extraction ← Constraint Resolver
       ↓
[Entity Resolution → Cluster Merge → Signal Score]
       ↓
[Niche Report] ← GLM Synthesizer ← Gate Answers + Signal Data
```

---

## Data Flow

1. User invokes `/discover` command in Discord (DM or channel).
2. Bot initializes a new discovery session and presents Gate 1: "What verticals interest you? (Select all that apply)" with interactive buttons.
3. User selects verticals (e.g., "DevTools, Productivity, Finance").
4. Bot runs signal scan for selected verticals, returns Gate 2: "What's your technical stack? (Frontend, Backend, Full-stack, No-code)" with signal-informed context.
5. User answers; bot narrows scan parameters and runs validation scan.
6. Gate 3: "B2B or B2C? What price range?" — bot shows signal data for each direction.
7. User answers; bot runs competitive density scan for the narrowed niche space.
8. Gate 4: "What's your timeline? (Weekend, 30 days, 90 days)" — bot shows time-adjusted opportunity scores.
9. User answers; bot generates final Niche Report using GLM synthesizer.
10. Bot delivers the Niche Report as an embedded Discord message with ranked opportunities, Signal Scores, quotes, and 30-day launch plans.

## Tech Stack

- **Bot Framework**: discord.js v14 for Discord API integration
- **Runtime**: Node.js with TypeScript
- **AI/Generation**: z.ai GLM API for niche report synthesis and gate question refinement
- **Signal Processing**: last30days skill (signal scanner, interactive gate, entity resolution, cluster merge, validation)
- **State Management**: SQLite via better-sqlite3 for per-user session state
- **Hosting**: Railway or Fly.io for always-on bot process
- **Database**: Prisma ORM for session persistence and discovery history
- **Deployment**: Docker container with automatic restart
