## Bug Fixes During Development

### Cover page absolute positioning in multi-page PDF
- **Root cause**: `html2pdf-next.js` converts `position: absolute` elements to static flow for multi-page documents, which destroyed the cover page layout where elements like the badge banner, ghost number, and brand logos needed precise positioning
- **Fix**: Separated cover page into a standalone HTML file, rendered with `html2poster.js` (single-page renderer that preserves absolute positioning), then merged with content PDF using `pypdf` page merge
- **How it was fixed**: Created `cover.html` as a separate file with poster-style layout; used `html2poster.js --width 210mm` for cover; used `html2pdf-next.js` for content pages only; merged via `pdf.py pages.merge`
- **Result**: Cover page renders correctly with all elements in their intended positions

### Footer overlap on content pages
- **Root cause**: `position: absolute; bottom: 20px` footers overlapped with content when page content varied in length across different pages. On shorter pages, the footer appeared in the middle of content; on longer pages, it was pushed below the visible area
- **Fix**: Changed footer from absolute positioning to document flow. Added `padding-top: 12px`, `margin-top: 20px`, and `border-top: 1px solid rgba(45, 45, 94, 0.3)` so it naturally follows content
- **How it was fixed**: Replaced `.page-footer { position: absolute; bottom: 20px; }` with `.page-footer { padding-top: 12px; margin-top: 20px; border-top: ... }`
- **Result**: Footers appear consistently after content without overlap on any page
